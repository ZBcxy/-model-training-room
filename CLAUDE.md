# 模型训练室 (Model Training Room)

## 项目定位

一款 **本地化模型微调一站式工具**，让用户通过图形界面完成从模型下载、数据准备、微调训练到评估导出的全流程，无需手写代码或手动配置 YAML。

### 核心价值

> 把微调模型这件事，从「读文档 + 写脚本 + 调 yaml + 看 traceback」变成「挑选 + 配置 + 点击 + 等待 + 使用」。

### 目标用户

有基础技术能力但不想折腾环境的开发者/爱好者。不是纯小白（会用终端），但也不想每次都去翻 Hugging Face 文档、调 LLaMA-Factory 配置、手动算显存。

### 产品形态

**Web 界面**（Gradio 原型 → 后续可升级为 React + FastAPI）

---

## 技术栈（已确定）

| 层面 | 选择 | 理由 |
|------|------|------|
| 后端 | Python FastAPI | ML 生态唯一选择 |
| 前端 | Gradio（MVP） | ML 项目最快速的原型框架 |
| 模型下载 | huggingface_hub + modelscope SDK | 覆盖国内外两大模型源 |
| 微调引擎 | LLaMA-Factory（调用其 Python API） | 社区最成熟，支持数百种模型，LoRA/QLoRA 全支持 |
| 数据集搜索 | Hugging Face Datasets + ModelScope Datasets + OpenDataLab | 三大数据集平台 |
| 导出 | llama.cpp 的 convert + quantize | GGUF 导出适配 Ollama 本地推理 |
| 数据库 | SQLite | 零配置，存模型缓存记录、训练历史、实验管理 |

## 功能架构

### 阶段 0：环境就绪检查

- **硬件检测**：自动检测 GPU 型号/显存/CUDA 版本/内存/磁盘，告知用户能跑多大的模型
- **环境依赖检查**：PyTorch、CUDA toolkit、编译工具链是否就绪，缺什么一键安装
- **显存预算计算器**：输入模型参数量 + 微调方式，预估所需显存，防止训练中途 OOM

### 阶段 1：模型中心（📦）

- **多源搜索**：统一搜索 Hugging Face + ModelScope + GitHub
- **许可证自动标注**：🟢 可商用 / 🟡 需审查 / 🔴 不可商用
- **模型兼容性检查**：检查架构是否被 LLaMA-Factory 支持，chat template 是否标准
- **断点续传下载**：大模型动辄几十 GB，网络中断不重头来
- **本地模型库**：已下载模型清单、版本管理、去重、清理
- **离线模式**：模型已在本地时无需联网

### 阶段 2：数据准备（🗂️）—— 核心竞争力

#### 三条数据获取路径：

**路径 1：📚 内置精选数据集库（零门槛）**
- 10-15 个精选数据集，按场景分类（对话、代码、翻译、客服等）
- 每个数据集展示：场景图标、条数、大小、许可证、前 3 条预览
- 一键使用

**路径 2：🔍 在线数据搜索（中门槛）**
- 搜索 Hugging Face Datasets + ModelScope 数据集
- 结果展示：平台来源、许可协议、条数、大小、内容预览
- 一键下载并导入

**路径 3：🪄 数据生成向导（零门槛，杀手功能）**

- **方式 A：手写少量示例 → AI 批量扩充**：用户写 3-5 个完美示例，系统用大模型 API 自动扩充到指定数量
- **方式 B：上传文档/知识库 → 自动生成问答对**：上传 PDF/Word/Markdown/网页链接，自动生成训练数据
- **方式 C：自然语言描述需求 → 自动搜数据 + 采样**：描述需求，系统自动去 HF/ModelScope 搜、合并、去重、采样到目标数量

#### 数据管理与教育功能：

- **格式自动转换**：JSON/CSV/纯文本 → 自动转 ShareGPT/Alpaca/对话格式
- **数据预览**：表格视图 + 对话视图
- **一键清洗**：去重、去空、截断
- **质量报告**：长度分布、语言检测、重复度
- **内置提示模板**：客服、代码助手、翻译、摘要等场景的 prompt 模板
- **训练/验证集自动切分**：支持分层采样
- **数据认知指南**：数量建议、格式示例、好数据 vs 坏数据对比、清洗检查清单

### 阶段 3：训练配置（🧪）

- **三步向导**：选模型 → 配数据 → 调参数 → 开始
- **微调方式选择**：QLoRA (>6GB)、LoRA (>16GB)、全参微调 (>40GB)，根据实际显存自动推荐
- **智能参数推荐**：根据模型大小和显存自动推荐 LoRA rank、batch size、learning rate、epochs
- **预设方案**：快速尝试 (1h) / 标准微调 (3h) / 深度训练 (8h) / 自定义
- **Chat Template 自动检测与对齐**
- **显存预算可视化**：模型权重 / 优化器状态 / 激活值 / 余量，分项展示
- **高级参数默认折叠**：LoRA Alpha、Dropout、Optimizer、Scheduler、Warmup、Flash Attention 等
- **配置文件最终预览**：确认后再启动

### 阶段 4：训练监控（🔬）

- **实时 Loss 曲线 + 学习率变化**
- **系统状态**：GPU 使用率、显存占用、温度、内存
- **训练信息**：当前 Epoch/Step、Token/秒、剩余时间预估
- **实时输出预览**：每 N step 采样一次模型输出，直观感受训练进展
- **操作按钮**：暂停、停止、保存 Checkpoint，停止时自动保存

### 阶段 5：评估与导出（🏆）

- **对话式测试**：在界面中直接与微调后模型对话
- **A/B 对比**：相同问题，原始模型 vs 微调模型并排对比
- **GGUF 导出**：量化级别选择（q4_0 / q4_K_M / q5_K_M / q8_0 / fp16），预估文件大小
- **量化工具集成**：llama.cpp 的 convert + quantize
- **Model Card 自动生成**：按 Hugging Face 标准生成模型说明卡片

### 阶段 6：实验管理（📊）

- **历史记录表**：所有训练任务的列表，含模型、数据、Loss、时间、状态
- **单次详情**：完整参数记录、Loss 曲线、Checkpoint 列表
- **操作**：重新训练、导出 GGUF、复制配置、删除

## 全局 UI 设计

### 布局：三栏式

```
┌──────────────────────────────────────────────────────────┐
│  顶栏：Logo + 设置 + 深色模式切换                   48px │
├────────┬──────────────────────────────────┬──────────────┤
│ 左侧   │                                │   右侧        │
│ 导航   │        主工作区                  │   任务面板    │
│ 6项    │   （随导航切换内容）              │   (可折叠)   │
│ 固定   │                                │             │
├────────┴──────────────────────────────────┴──────────────┤
│  底部状态栏：GPU · 显存 · 磁盘 · 运行中任务               │
└──────────────────────────────────────────────────────────┘
```

### 左侧导航（6 项）

1. 📦 模型中心 —— 搜索/浏览/下载/本地库
2. 🗂️ 数据准备 —— 内置库/搜索/生成向导/导入/预览
3. 🧪 训练配置 —— 选模型→配数据→调参数→确认
4. 🔬 训练监控 —— Loss曲线/系统状态/实时输出
5. 🏆 评估导出 —— 对话测试/A-B对比/GGUF导出
6. 📊 实验管理 —— 历史记录/详情/对比

### UI 设计原则

- **始终可见的状态栏**：底部常驻系统资源信息
- **进度可感知**：任何超过 5 秒的操作都有进度条 + 预估时间 + 速度
- **非破坏性操作**：停止训练自动保存 checkpoint，删除前需确认
- **智能默认值**：所有参数都有推荐值，高级选项默认折叠
- **错误友好**：OOM 不是 traceback，而是「显存不足，建议：① 降低 batch size ② 使用 QLoRA」
- **响应式**：窗口缩小到 1024px 时，左侧导航收成图标，右侧任务面板折叠

## 设计决策记录

### 为什么不用 Electron 做桌面应用？
MVP 阶段用 Gradio（Web UI），开发速度最快。后续可升级为 React + FastAPI 的正式 Web 应用，或 Electron 套壳做桌面应用。

### 为什么用 LLaMA-Factory 而不是自己封装 Transformers？
LLaMA-Factory 已经解决了模型兼容性、训练模板、数据格式适配等几百个坑，自己封装工作量大且易出错。调用其 Python API 即可。

### 为什么数据生成向导是核心竞争力？
大多数微调工具只解决「怎么训」，不解决「训什么」。数据准备是微调最费时最劝退的环节，提供零门槛的数据获取方案是差异化优势。

### 为什么许可证标注这么重要？
开源模型协议复杂（Apache 2.0、MIT、CC、Llama License 等），普通用户难以判断是否可以商用。自动标注 + 风险提示避免法律纠纷。
